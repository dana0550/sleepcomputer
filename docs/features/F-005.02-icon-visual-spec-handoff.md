# [F-005.02] Icon Visual Spec and Handoff

## Summary

Define icon geometry, readability constraints, and delivery artifacts used by design-to-engineering handoff.

## Goals

- Keep icon semantics clear at small menu-bar sizes.
- Keep handoff file names and paths stable.
- Keep UI ON/OFF visual behavior aligned with design spec.

## Non-Goals

- Runtime design tooling integration.
- Multi-color menu-bar icon variants.

## Requirements

- R1: Visual spec must define 20x20 canvas, 19x19 live area target, and small-size readability constraints.
- R2: Handoff must preserve required filenames for off/open/closed SVG files.
- R3: Repo must include delivery metadata files (`ICON_ASSET_SPECS.csv`, `ICON_MANIFEST.json`, `MENU_COPY.csv`).
- R4: Menu row ON-state visual semantics must use explicit blue circle + white icon treatment.

- AC1: Design handoff checklist can be executed using in-repo visual spec guidance.
